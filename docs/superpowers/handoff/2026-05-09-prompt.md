# Handoff prompt — paste into a fresh Claude session

Copy everything between the `--- BEGIN PROMPT ---` and `--- END PROMPT ---` markers into a new Claude Code conversation in this directory. The prompt is self-contained: it points Claude at the right files and tells it where Tim is in the workflow.

--- BEGIN PROMPT ---

I'm Tim, working on the SYNTHERION devlog site at `C:\coding\syntherion-website`. The site is built (16-task implementation complete, deployed to GitHub Pages, live at https://tismnd.github.io/syntherion-website/). I'm picking back up from a previous Claude session.

Before doing anything else, read these files in order:

1. `docs/superpowers/handoff/2026-05-09-handoff.md` — current project state, what's done, what's pending out-of-band, deviations from the original plan, and where I am in the workflow.
2. `docs/superpowers/specs/2026-05-08-syntherion-devlog-site-design.md` — design spec, editorial direction, story nuggets and voice samples I've shared.

After reading those: the dev server should already be running on `http://localhost:8080/` (live-reload Eleventy). If it's not, restart with `npm run dev` from the repo root. Don't browse the github.io URL — it renders unstyled because of project-page subpath behavior; localhost is the canonical preview surface.

Where I am right now: I want visuals locked before any post drafting. I have 27 PNGs in `content/intake/` waiting, plus more writing nuggets I want to share — but post drafting is on hold until I say go. Your first message back should:
1. Confirm you've read the three files.

--- END PROMPT ---

## How to use it

1. Open a new Claude Code session in `C:\coding\syntherion-website`.
2. Paste the prompt above as your first message.
3. Claude reads the three reference docs and reports back its understanding.
4. From there, work as normal.

## What this prompt deliberately doesn't include

- The full content of the spec / plan / context doc — those are in the files. The prompt is the pointer, not a re-summary. Re-summarizing would drift from source of truth and waste tokens.
- Specific URLs / IDs / tokens — those live in the handoff doc and in `src/_data/site.js`.
- Step-by-step instructions for any specific task — Claude reads the spec/plan/handoff and can figure out what's next based on conversation.
