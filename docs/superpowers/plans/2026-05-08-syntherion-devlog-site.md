# SYNTHERION Devlog Site Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Convert `syntheriongame.com` from a single-page promo into an Eleventy-powered devlog site with a hybrid landing page, three post types (origins / tools / devlog), persistent Steam + Discord CTAs, a real Buttondown mailing-list signup, Cloudflare Web Analytics, and a self-hosted hero trailer on Cloudflare R2.

**Architecture:** Static site generated by Eleventy from markdown posts. Output deploys to GitHub Pages (existing CNAME). Hero video served from Cloudflare R2 (free tier). Mailing list via Buttondown form. Analytics via Cloudflare Web Analytics. No backend, no Firebase.

**Tech Stack:** Node 20+, Eleventy 3.x, Nunjucks templates, vanilla CSS (adapted from existing `style.css`), Cloudflare R2 (video hosting), Buttondown (mailing list), Cloudflare Web Analytics.

- [ ] **Step 1: Add posts directory data file**

`src/posts/posts.11tydata.js` — defaults applied to every markdown file in `src/posts/`:

```js
module.exports = {
  layout: "layouts/post.njk",
  tags: ["posts"],
  permalink: "/devlog/{{ page.fileSlug }}/",
  eleventyComputed: {
    // Hide unpublished posts from production builds.
    // In dev (`eleventy --serve`), ELEVENTY_RUN_MODE === "serve" and we keep them visible.
    eleventyExcludeFromCollections: data => {
      if (process.env.ELEVENTY_RUN_MODE === "serve") return false;
      return data.published !== true;
    }
  }
};
```

The pattern: in dev, all posts (drafts included) appear in collections so you can preview them. In a production build (no `--serve`), only `published: true` posts make it into the feed.

- [ ] **Step 2: Add a published-filter to the posts collection**

Modify `.eleventy.js` to add a filtered collection. Replace the file with:

```js
module.exports = function(eleventyConfig) {
  eleventyConfig.addPassthroughCopy({ "src/styles": "styles" });
  eleventyConfig.addPassthroughCopy({ "src/assets": "assets" });
  eleventyConfig.addPassthroughCopy("CNAME");

  // All posts (already filtered by published flag in posts.11tydata.js)
  eleventyConfig.addCollection("posts", (collectionApi) => {
    return collectionApi
      .getFilteredByTag("posts")
      .sort((a, b) => b.date - a.date);
  });

  // Subset collections per type
  for (const type of ["origins", "tools", "devlog"]) {
    eleventyConfig.addCollection(type, (collectionApi) => {
      return collectionApi
        .getFilteredByTag("posts")
        .filter(p => p.data.postType === type)
        .sort((a, b) => b.date - a.date);
    });
  }

  // readableDate filter for templates
  eleventyConfig.addFilter("readableDate", (d) => {
    return new Date(d).toLocaleDateString("en-US", {
      year: "numeric", month: "long", day: "numeric"
    });
  });

  return {
    dir: {
      input: "src",
      output: "_site",
      includes: "_includes",
      data: "_data"
    },
    templateFormats: ["njk", "md", "html"],
    markdownTemplateEngine: "njk",
    htmlTemplateEngine: "njk"
  };
};
```

## Task 16: GitHub Pages deploy

**Goal:** Get the built `_site/` published to `syntheriongame.com`. GitHub Pages does NOT run Node by default — it serves whatever's in the configured branch/folder. Two approaches: commit `_site/` to a branch, or use a GitHub Actions workflow.

**Recommended: GitHub Actions** — keeps the build off the main branch and stays in `git push` workflow.

**Files:**
- Create: `.github/workflows/deploy.yml`

- [ ] **Step 1: Verify current GH Pages config**

Visit the repo's Settings → Pages. Check current source. The CNAME in the repo confirms `syntheriongame.com` is the configured domain.

- [ ] **Step 2: Create the deploy workflow**

`.github/workflows/deploy.yml`:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - run: npm ci
      - run: npm run build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: _site

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - id: deployment
        uses: actions/deploy-pages@v4
```

- [ ] **Step 3: Switch GH Pages source**

Repo Settings → Pages → Build and deployment → Source = **GitHub Actions**.
